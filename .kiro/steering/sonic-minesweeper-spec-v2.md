> **목적**: Claude Code + cc-sdd 학습용 프로젝트.
> 이 문서를 `spec-init`의 입력으로 사용하여 requirements → design → tasks 흐름을 진행합니다.

## 1. 프로젝트 개요

**소리로 지뢰를 탐지하는 지뢰찾기 게임.**

각 지뢰가 고유 주파수의 소리를 가지며, 셀을 롱프레스하면 주변 지뢰들의 소리가 거리에 따라 감쇄되어 중첩 재생된다. 플레이어는 소리 단서를 분석하여 지뢰 위치를 탭으로 지목한다. 리얼타임으로 흐르는 시간이 유일한 자원이며, 탐색과 오답에 시간 비용이 부과된다.

### 핵심 컨셉

- **듣기(롱프레스)와 찍기(탭)** 두 가지 행동만 존재
- 시간 = 생명 = 자원: 리얼타임 카운트다운 + 행동 시 추가 차감
- 정보 수집(탐색)에도 비용 → "언제 듣고, 언제 찍을 것인가"의 전략적 판단

### 기술 스택

| 범주 | 선택 |
|------|------|
| 프레임워크 | React 19 + Vite |
| 스타일링 | TailwindCSS v4 (`@import "tailwindcss"`, `@theme` 블록) |
| 오디오 | Web Audio API (OscillatorNode + GainNode, 라이브러리 없음) |
| 상태관리 | useReducer + Context (외부 라이브러리 없음) |
| 빌드/배포 | Vite → 정적 호스팅 (Vercel / GitHub Pages 등) |

## 2. 오디오 시스템

### 2.1 주파수 할당

- **범위**: 220Hz ~ 1760Hz (A3 ~ A6, 3옥타브)
- **배분 방식**: 로그 스케일 (사람의 음감은 로그적)
- **공식**: `frequency = 220 × 2^(i / mineCount × 3)` (i = 0 ~ mineCount-1)
- **최소 간격 보장**: 인접 주파수 간 최소 반음(약 5.9%) 이상 차이
- **게임 시작 시 셔플**: 주파수 배열을 무작위 섞어 지뢰에 할당 → 매 게임마다 다른 소리 조합

### 2.2 거리 감쇄 모델

- **거리 계산**: 유클리드 거리 (`d = √((x1-x2)² + (y1-y2)²)`)
- **최대 감지 반경**: 난이도별 차등 (§3.3 참조)
- **감쇄 공식**: 역제곱 기반
  ```
  gain = maxGain × (1 - d / maxRadius)²     (d ≤ maxRadius)
  gain = 0                                    (d > maxRadius)
  ```
- **maxGain**: 0.3 (클리핑 방지, 중첩 시에도 합산 gain ≤ 1.0 되도록)
- **중첩**: 반경 내 모든 지뢰의 OscillatorNode를 개별 GainNode로 연결 → 하나의 destination에 합산

### 2.3 소리 재생 규칙

| 상황 | 소리 | 길이 |
|------|------|------|
| **롱프레스 (탐색)** | 감지 반경 내 지뢰 중첩음 | fade-in 0.05s, fade-out 0.3s, 누르고 있는 동안 계속 재생 |
| **정답 탭** | 딩동댕 소리 |  |
| **오답 탭** | 땡 | |
| **게임오버** | 콰광 소리 | 전체 약 2초 |
| **승리** | 승리 팡파르 (전 주파수 화음) | 1초 |

### 2.4 모바일 오디오 제약 대응

- 게임 시작 화면에 **"탭하여 시작"** 버튼 → 이 인터랙션으로 AudioContext 생성/resume
- AudioContext는 앱 전역에서 단일 인스턴스로 관리 (Suspended → Running 상태 전환)
- 이어폰 사용 권장 안내 메시지 (첫 게임 시 한 번 표시)

## 3. 게임 메커닉

### 3.1 핵심 인터랙션

| 제스처 | 동작 | 시간 비용 |
|--------|------|-----------|
| **롱프레스** (≥ 500ms) | 소리 탐색 — 해당 셀 기준으로 반경 내 지뢰 중첩음 재생 | **차감됨** (§3.4) |
| **탭** (< 300ms) | 지뢰 지목 — 지뢰면 정답, 빈 셀이면 오답 | 정답: **무료** / 오답: **대량 차감** (§3.4) |

- 롱프레스 진행 표시: 셀 주변에 원형 progress indicator
- 이미 정답 처리된 셀(깃발)은 탭/롱프레스 불가
- **셀 상태**: `hidden` | `flagged`(정답) | `exploded`(오답 3회 시 지뢰 공개) | `revealed`(§3.2 참조)

### 3.2 셀 공개 규칙
정답 → 깃발만 표시, 셀 공개 없음 → 순수 소리 의존, 하드코어

### 3.3 난이도 레벨

| 레벨 | 보드 크기 | 지뢰 수 | 감지 반경 | 시간 예산 | 롱프레스 비용 | 오답 비용 |
|------|-----------|---------|-----------|-----------|---------------|-----------|
| **초급** | 8 × 8 | 6 | 6.0 | 120초 | -3초 | -30초 |
| **중급** | 12 × 12 | 20 | 5.0 | 240초 | -4초 | -50초 |
| **고급** | 16 × 16 | 40 | 4.0 | 420초 | -5초 | -70초 |

- 고급에서 감지 반경이 줄어들어 소리 단서가 더 제한적
- 정답 탭은 모든 난이도에서 시간 비용 0
- 난이도 선택은 게임 시작 화면에서

### 3.4 시간 시스템 (핵심 자원)

```
┌─────────────────────────────────────────────────────────┐
│  시간 = 리얼타임 카운트다운 + 액션별 추가 차감            │
│                                                          │
│  매 프레임: remainingTime -= deltaTime                   │
│  롱프레스 완료 시: remainingTime -= listenCost            │
│  오답 탭 시: remainingTime -= wrongTapCost                │
│  정답 탭 시: 차감 없음 (보상)                             │
└─────────────────────────────────────────────────────────┘
```

- 시간은 소수점 1자리까지 표시 (예: 87.3초)
- 10초 이하: 빨간색 경고 + 깜빡임
- 5초 이하: 긴급 경고음 (짧은 비프 0.5초 간격)

### 3.5 게임오버 조건 (이중)

| 조건 | 트리거 |
|------|--------|
| **시간 소진** | remainingTime ≤ 0 |
| **오답 3회 누적** | wrongCount ≥ 3 |

- 어느 조건이든 먼저 달성되면 즉시 게임오버
- 게임오버 시: 모든 지뢰 위치 공개 + 순차 아르페지오 재생
- 오답 셀은 빨간 × 표시로 강조

### 3.6 승리 조건

- 모든 지뢰를 정확히 탭하면 승리
- 시간이 남아있고 오답이 3회 미만인 상태에서 마지막 지뢰를 맞추면 즉시 승리

### 3.7 점수 시스템

```
최종 점수 = 기본 점수 + 시간 보너스 + 연속 보너스 - 오답 감점

기본 점수:
  정답 1개 = +200점

오답 감점:
  오답 1회 = -100점

시간 보너스:
  남은 시간(초) × 10
  → 빠르게 클리어할수록, 탐색을 적게 할수록 높은 점수

연속 보너스:
  연속 정답 N개 = N × (N+1) / 2 × 50 (삼각수)
  예: 연속 3개 = 3×4/2×50 = 300점
  오답 시 연속 카운터 리셋
```

- **게임오버 시**: 기본 점수의 50%만 인정, 보너스 없음
- **로컬 하이스코어**: 난이도별 상위 5개 기록 (localStorage)

**점수 시뮬레이션 (초급 이상적 플레이):**
```
지뢰 6개 전부 정답: 6 × 200 = 1,200
연속 6개 보너스: 6×7/2×50 = 1,050
남은 시간 80초: 80 × 10 = 800
오답 0회: 0
총점: 3,050점
```

## 4. 모바일 UX

### 4.1 레이아웃 & 반응형

- **최소 셀 크기**: 44 × 44px (Apple HIG 터치 타겟)
- **초급 (8×8)**: 셀이 화면 너비에 꽉 차도록 자동 조절 — 스크롤 불필요
- **중급 (12×12)**: 가로 기준 fit, 세로 약간 스크롤 가능
- **고급 (16×16)**: 핀치 줌 + 패닝 지원 (CSS transform 기반)
- **상단 바**: 잔여 시간 | 남은 생명(♥♥♥) | 현재 점수 (항상 고정)
- **하단**: 새 게임 / 난이도 변경 버튼

### 4.2 시각적 피드백

| 이벤트 | 시각 피드백 |
|--------|------------|
| **롱프레스 중** | 셀 주변 원형 progress ring (500ms 채워짐) |
| **소리 재생 중** | 동심원 파동 애니메이션 (감지 반경만큼 퍼짐) |
| **무음 탐색** | "범위 내 지뢰 없음" 툴팁 + 반경 표시 (반투명 원) |
| **소리 세기 힌트** | 파동 색 채도로 총 볼륨 표현 (클수록 진한 색) |
| **정답 탭** | 초록 체크마크 + pulse 애니메이션 + 지뢰 아이콘 |
| **오답 탭** | 빨간 × + shake 애니메이션 + 생명 하트 깜빡임 |
| **시간 차감** | 시간 표시 옆에 "-3s" 같은 플로팅 텍스트 (빨간색, 위로 사라짐) |
| **게임오버** | 지뢰 순차 공개 (0.1초 간격) + 화면 어두워짐 |
| **승리** | confetti 효과 (CSS, 라이브러리 없음) |
| **10초 이하** | 타이머 빨간색 + 깜빡임 |

### 4.3 접근성

- 시각적 보조 모드 (토글): 열린 셀에 인접 지뢰 수 크게 표시, 소리 세기를 색상 그라데이션으로 표시
- 키보드 네비게이션 (데스크톱): 화살표 이동, Enter 탭(지목), Space 롱프레스(탐색)
- 다크모드: 시스템 설정 연동 (`prefers-color-scheme`)

## 5. 화면 구성

### 5.1 시작 화면

- 게임 타이틀 + 간단한 설명 ("소리로 지뢰를 찾아라")
- 난이도 선택 (3개 카드 — 각각 보드 미리보기 + 시간/지뢰 수 표시)
- **"탭하여 시작"** 버튼 (AudioContext 활성화 트리거)
- 이어폰 사용 권장 안내 (첫 방문 시)
- 하이스코어 보기 링크

### 5.2 게임 플레이 화면

```
┌──────────────────────────────────┐
│  ⏱ 87.3s    ♥♥♡    ⭐ 1,200     │  ← 상단 고정 바
├──────────────────────────────────┤
│                                  │
│         [ 게임 보드 ]            │  ← 중앙 (가능한 화면 중앙)
│                                  │
├──────────────────────────────────┤
│     [새 게임]  [난이도 변경]      │  ← 하단
└──────────────────────────────────┘
```

- 소리 재생 중 스피커 아이콘 진동
- 남은 지뢰 수 표시: "🎯 4/6" (찾은 수 / 전체)

### 5.3 결과 화면 (모달)

- 승리/패배 상태 + 원인 (시간 소진 / 오답 3회)
- 점수 상세 내역 (기본 + 시간 보너스 + 연속 보너스 - 감점)
- 하이스코어 갱신 시 강조
- [다시 하기] [난이도 변경] [메인으로]

## 6. 비기능 요구사항

### 6.1 성능

- 첫 로드 번들 사이즈: < 200KB (gzip)
- 셀 액션 → 소리 재생 지연: < 50ms
- 리얼타임 타이머 정확도: requestAnimationFrame 기반, ±16ms 이내
- 16×16 보드 렌더링 시 60fps 유지

### 6.2 브라우저 지원

- Chrome 90+ (데스크톱/모바일)
- Safari 15+ (iOS/macOS) — Web Audio API 제약 대응
- Samsung Internet 15+
- Firefox 90+
